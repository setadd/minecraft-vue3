# Minecraft 复刻项目 - 功能规格说明书

## 1. 项目概述

### 1.1 项目定位
基于 Vue3 + Three.js 的 Web 版 Minecraft 沙盒建造游戏，第一阶段聚焦核心玩法：**放置/破坏方块**。

### 1.2 技术栈
| 层级 | 技术选型 |
|------|---------|
| 框架 | Vue 3.4+ (Composition API) |
| UI 库 | Element Plus |
| 3D 引擎 | Three.js r150+ |
| 语言 | TypeScript 5.x |
| 构建 | Vite 5.x |
| 状态管理 | Pinia |
| 本地存储 | IndexedDB (idb-keyval) |

---

## 2. 核心玩法

### 2.1 玩家交互
| 操作 | 输入 | 说明 |
|------|------|------|
| 移动 | W/A/S/D | 前后左右移动 |
| 跳跃 | Space | 跳跃（仅当在地面时） |
| 视角 | 鼠标移动 | 第一人称视角旋转 |
| 锁定指针 | 鼠标左键（游戏内） | 进入/退出指针锁定 |
| 破坏方块 | 鼠标左键（长按） | 挖掘方块（有进度条） |
| 放置方块 | 鼠标右键 | 在选中位置放置方块 |
| 快捷选择 | 数字键 1-9 | 选择快捷栏方块 |
| 快捷栏循环 | Q | 切换快捷栏选中位置 |
| 暂停菜单 | Esc | 打开/关闭暂停菜单 |

### 2.2 交互限制
| 限制项 | 数值/规则 |
|--------|----------|
| 最大挖掘距离 | 10 格（从玩家位置计算） |
| 最大放置距离 | 10 格（从玩家位置计算） |
| 放置冲突检测 | 禁止在玩家碰撞体内放置 |
| 点击限流 | 100ms 间隔（防连点） |
| 边界处理 | 空气墙（世界边界不可穿越） |
| 玩家卡住处理 | 自动弹出到最近安全位置 |

## 3. 世界系统

### 3.1 世界配置
| 参数 | 默认值 | 说明 |
|------|--------|------|
| 世界大小 | 可配置 | 默认 256x256 区块 |
| 区块大小 | 16x16x256 | 标准 Minecraft 区块 |
| 渲染距离 | 4-8 区块 | 可配置，默认 6 |
| 海平面高度 | Y=64 | 海洋生成基准 |

### 3.2 地形生成
使用 Perlin/Simplex 噪声生成：

| 地形类型 | 特征 | 生成概率 |
|---------|------|---------|
| 平原 | 平坦地势，Y=60-70 | 40% |
| 山脉 | 高耸地势，Y=80-120 | 30% |
| 海洋 | 低洼水域，Y=50-60 | 30% |

### 3.3 洞穴系统
- **第一阶段：不实现**
- 预留接口：`WorldGenerator.generateCaves()`

### 3.4 方块系统

#### 3.4.1 基础方块列表（10 种）
| ID | 名称 | 特性 | 挖掘时间 (秒) |
|----|------|------|--------------|
| 1 | 草方块 | 固体，不透明 | 1.0 |
| 2 | 泥土 | 固体，不透明 | 1.0 |
| 3 | 石头 | 固体，不透明 | 1.5 |
| 4 | 木头 (橡木) | 固体，不透明 | 1.0 |
| 5 | 树叶 (橡木) | 固体，半透明 | 0.5 |
| 6 | 沙子 | 固体，受重力 | 0.5 |
| 7 | 玻璃 | 固体，透明 | 0.3 |
| 8 | 砖块 | 固体，不透明 | 1.0 |
| 9 | 煤矿 | 固体，不透明 | 2.0 |
| 10 | 基岩 | 固体，不可破坏 | ∞ |

#### 3.4.2 方块数据结构
```typescript
interface BlockDef {
  id: number
  name: string
  texture: string | string[] // 单纹理或 [上,下,左,右,前,后]
  transparent: boolean
  solid: boolean
  gravityAffected: boolean
  hardness: number // 挖掘难度
  blastResistance: number // 爆炸抗性（预留）
}
```

## 7. 存档系统

### 7.1 存储方案
- 使用 **IndexedDB** 存储世界数据
- 每个世界独立存储
- 自动保存（每 30 秒或离开时）

### 7.2 存储数据结构
```typescript
interface SaveData {
  worldSeed: number
  worldName: string
  playerPosition: [number, number, number]
  playerRotation: [number, number]
  inventory: InventorySlot[]
  modifiedChunks: Map<string, Uint8Array> // 仅存储修改过的区块
  timestamp: number
}
```

### 7.3 存档操作
| 操作 | 触发 |
|------|------|
| 自动保存 | 每 30 秒 |
| 手动保存 | 暂停菜单中 |
| 读取存档 | 主菜单选择世界 |
| 删除存档 | 主菜单中 |

## 8. 性能要求

### 8.1 目标性能
| 指标 | 目标值 |
|------|--------|
| 帧率 | 60 FPS（渲染距离=6 时） |
| 加载时间 | 初始世界 < 5 秒 |
| 内存占用 | < 500MB |

### 8.2 优化策略
| 技术 | 说明 |
|------|------|
| 视锥体剔除 | 只渲染可见区块 |
| 背面剔除 | 不渲染被遮挡的面 |
| InstancedMesh | 同种方块批量渲染 |
| 几何体合并 | 相邻方块合并网格 |
| LOD | 远距离简化渲染（预留） |
| Web Worker | 世界生成异步化（预留） |

### 8.3 可配置选项
| 选项 | 档位 |
|------|------|
| 渲染距离 | 4 / 6 / 8 / 12 |
| 阴影质量 | 关 / 低 / 高 |
| 粒子效果 | 关 / 开 |
| 云渲染 | 关 / 开 |

| 模块 | 职责 |

## 11. 边缘场景处理

| 场景 | 处理方案 |
|------|---------|
| 玩家卡进方块 | 自动弹出到最近安全位置 |
| 放置方块与玩家重叠 | 禁止放置，显示提示 |
| 快速连续点击 | 100ms 限流 |
| 世界边界 | 空气墙阻挡 |
| 挖掘距离超限 | 不执行，显示距离提示 |
| 方块更新导致卡顿 | 分帧处理，每帧更新部分区块 |
| 内存过高 | 强制 GC，卸载远区块 |
| 存档损坏 | 备份机制，可回滚 |
